Dear students,

Thank you all for taking part in the first ever Data Science Lab competition!

There have already been some interesting activities on the leaderboard, with plenty of progress being made in very short amounts of time. Congratulations to all those who already submitted! Today, with the second group joining the lab,
we expect even more action on the leaderboard :)

Almost one day into the competition, there are some considerations we would like to make.

- We recommended using the os.listdir() function to read the contents of a directory. Please note that, if you are using this function, the files will not be sorted in any fashion. This means that, when you assign the labels in the submission
file, you should make sure that you are assigning the right Ids to the right labels. If you don't, you are essentially submitting a random solution (there is no correspondence between the labels and the id's). Random submissions have
scores that are close to 0, so make sure you are checking this detail when submitting!

- Remember that the public leaderboard is computed using only a portion of the labels you are submitting. The rest will be used for the final evaluation. So, don't give up just yet if you are close behind someone, you might actually score
better than them in the final leaderboard!

- There was a typo in the recommended TfidfVectorizer example. More specifically, the correct parameter is "stop_words=", not "stopwords=". If you are getting an error when running that code, make sure you are using the right syntax.
Also, in the same code, the variable name "documents" as passed to the __call__ function can be misleading: the __call__ method is executed on the entire collection one document at a time. However, the PDF file has been updated to fix
these issues.

- Make sure you submit your report (Exercise 2)! That will not only be needed to select the winner but exploring the clusters might also help you better understand them, thus improving your submissions. Don't forget to include a brief
description of the clustering approach you used

- The nltk library requires some additional data to work properly. Make sure you run this code beforehand (if you don't, the code will fail and ask you to run those instructions):
import nltk
nltk.download("stopwords")
nltk.download("punkt")
nltk.download("wordnet")

We wish you good luck for the (almost) 5 days of competition left!

Best regards,
Giuseppe Attanasio
